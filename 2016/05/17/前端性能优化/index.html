前端性能优化

发表于 2016-05-17 | 更新于 2018-09-16 | 分类于 性能优化

性能会影响用户体验，首屏出现缓慢、资源加载延迟、交互操作卡顿都会直接影响用户体验，说不定用户忍受不了，直接就关掉网站，同时，像Google也会将页面加载速度作为SEO的权重之一，所有前端性能也很重要。常见的优化手段有以下这些。

请求优化

  1. 合并CSS、JavaScript
  2. 将小图片合并成Sprite图
  3. 避免空请求、404请求和重定向
  4. 将静态资源划分到多个域上
  5. 按需加载
  6. 延迟加载
  7. 缓存一切可以缓存的资源和数据
  8. 使用CDN缓存
  9. 使用外联式CSS和JavaScript
  10. 减小Cookie大小

资源优化

  1. 压缩HTML、CSS、JavaScript
  2. 启动gzip

图片优化

  1. 使用CSS3、IconFont、SVG替代图片
  2. webp优于jpg，png8优于gif
  3. 使用srcset按需加载图片
  4. 延迟加载图片

HTML优化

  1. 减少DOM节点
  2. 使用伪元素替代DOM节点
  3. 将CSS放在头部
  4. 将js放在底部

CSS优化

  1. 避免使用Import
  2. 避免使用CSS表达式
  3. 避免嵌套过深
  4. 提取公共样式为公共类
  5. 使用transform、animation替代修改left、margin进行动画

JavaScript优化

  1. 使用事件代理
  2. 缓存DOM和计算结果
  3. 减少重排和重绘
  4. 避免使用setTimeout和setInterval，使用requestAnimationFrame替代
  5. 通过节流和防抖限制高频事件触发

总结

性能优化是一个巨大的工程问题，以上优化只是常规的优化手段，在具体的场景中需要根据性能指标进行进一步的优化，比如针对白屏时间、首屏时间、可交互时间、页面下载总时间的指标，需要不同的优化。
