常见的web攻击

发表于 2016-07-16 | 更新于 2018-09-16 | 分类于 web安全

常见的web攻击方式有XSS、CRSF等。

XSS

什么是XSS？

XSS，Cross-Site-Scripting，跨站脚本攻击。攻击者在网页上发布包含攻击性代码的数据，当浏览者看到此网页时，特定的脚本就会以浏览者用 户的身份和权限来执行。通过XSS可以比较容易地修改用户数据、窃取用户信息，以及造成其它类型的攻击。

XSS的原理及攻击方式

恶意攻击者往Web页面里插入恶意html代码，当用户浏览该页之时，嵌入其中Web里面的 (html代码/javascript代码) 会被执行，从而达到恶意攻击用户的特殊目的。

XSS漏洞出现的原理主要是：

  • 没有对用户输入的内容进行过滤，没有对输出进行转码
  • 没有区分内容是可展示数据还是恶意代码

因此，XSS容易出现在一下场景：

  • 搜索框
  • 留言区域

攻击方式：

  • 非持久型：攻击是一次性的，仅对当次的页面访问产生影响；
  • 持久型：会把攻击者的数据存储在服务器端，攻击行为将伴随着攻击数据一直存在。

防御方法

对用户输入的html标签以及常用漏洞标签属性、引号等进行过滤、转码。

CRSF

什么是CRSF？

CRSF，Cross Site Request Forgery，跨站请求伪造。通过伪造连接请求在用户不知情的情况下，让用户以自己的身份来完成攻击者需要达到的一些目的。

CRSF的原理

CRSF的攻击步骤主要是：
1、用户访问受信任的网站A并登录；
2、网站A验证通过，并在浏览器生成cookie；
3、用户在没有登出网站A的情况下，访问了恶意网站B；
4、恶意网站B返回恶意代码，并且发出请求要求访问网站A；
5、浏览器携带cookie，在用户未知的情况下，访问网站A。
6、网站A通过cookie判断，认为是用户进行的操作，处理请求，执行了网站B请求的恶意代码。

防御方法

  • 验证HTTP Referer字段：根据HTTP协议，在HTTP头中有一个字段叫Referer，它记录了该HTTP请求的来源地址。在通常情况下，访问一个安全受限页面的请求必须来自于同一个网站。
  • 在请求地址中添加token并验证
  • 每次请求操作都添加验证码
  • 在HTTP头中自定义属性并验证
